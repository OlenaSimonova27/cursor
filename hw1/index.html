Hello everybody!

  • git config --global user.name - это настройка имени пользователя, которая используется для настройки значений конфигурации на глобальном и локальном уровнях проекта
  • git config --global user.email - настройка электронной почты пользователя используемая на уровнях проекта
  • git init - настройка инициализирует новый репозиторий GIT и начинает отслеживание существующего каталога
  • git status - отображает состояние рабочего каталога и раздела проиндексированных файлов
  • git add -это первая команда в цепочке операций, предписывающей Git «сохранить» снимок текущего состояния проекта в истории коммитов
  • git commit - основные конструктивные элементы временной шкалы проекта Git, то есть делает снимок состояния проекта на текущий момент времени
  • git log - это настройка, которая перечисляет коммиты, сделанные в репозитории в обратном к хронологическому порядке "я-а"
  • git checkout - команда переключение между различными версиями целевого объекта
  • git branch - команда, которая позволяет создавать, просматривать, переименовывать и удалять ветки. Она не дает возможности переключаться между ветками или выполнять слияние разветвленной истории
  • git push - используется для выгрузки содержимого локального репозитория в удаленный репозиторий
  • git pull - используется для извлечения и загрузки содержимого из удаленного репозитория и немедленного обновления локального репозитория этим содержимым
  • git merge - выполняет слияние отдельных направлений разработки, созданных с помощью команды git branch , в единую ветку
  • git clone - утилита командной строки Git для выбора существующего репозитория и создания его клона
  • gitignore файлы - артефакты сборки и файлы, генерируемые машиной из исходных файлов в вашем репозитории, либо файлы, которые по какой-либо иной причине не должны попадать в коммиты
  • git revert - операция для безопасной отмены изменений, действие которой направлено в будущее
  • git reset - команда, используемая для отмены локальных изменений в репозитории Git
  • git rebase - один из способов в git, позволяющий объединить изменения двух веток
  • git cherry-pick - команда, с помощью которой можно выборочно применить коммиты Git к текущей рабочей ветке HEAD
  • git stash - позволяет на время «сдать в архив» (или отложить) изменения, сделанные в рабочей копии, чтобы вы могли применить их позже
